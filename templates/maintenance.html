M10-M19 Operations

New here? Follow these steps from top to bottom. Each step explains what to click and why.

Quick Start Order

  1. Start with Step 1 to check system health. This is read-only.
  2. Use Step 2 to build a work queue of records that may need fixes.
  3. Use Step 3 preview first, then apply changes only after you review the count.
  4. Use Step 4 to save a safety checkpoint before large edits.
  5. Use Step 5 to run optional jobs and review audit output.
  6. Use Step 6 to confirm safety settings (preview, approvals, checkpoint behavior).
  7. Use Step 7 only when approval tokens are required.

Step 1: Check System Health (M10-M12)

Click these three buttons left to right. They only inspect data and do not change records.

Step 2: Find Records to Fix (M13)

First run Drift Queue to load candidates, then mark those records with your queue marker.

Preview source: M13 Drift Queue. Undo path: M15 Restore Checkpoint.

Tip: Keep one marker format for all runs so it is easy to search and audit later.

Step 3: Preview, Then Apply Fixes (M14)

Run Preview first to see how many records would change. Apply stays disabled until preview finds candidates.

Only click Apply after you review the candidate count and confirm your safety settings in Step 6.

Step 4: Save a Restore Point (M15)

Create a checkpoint before large edits. Restore is for rollback if results are not what you expected.

Step 5: Run Jobs and Check History (M16-M17)

Load jobs, run the one you need, then load audit output to confirm what happened.

Step 6: Confirm Safety Rules (M18)

These settings control risk. Review them before any broad or destructive action, then save.

Step 7: Handle Approval Tokens (M19)

If two-step approval is enabled, request a token for your action, approve it, then verify in the list.
